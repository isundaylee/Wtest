VLAN Settings


The 802.1Q VLAN protocol on WMS5316 logically separates traffic on the same physical network. These settings apply only to the WMS5316.


Untagged VLAN

When checked, this option allows one VLAN to be configured as an "untagged VLAN". When WMS5316 sends frames associated with the untagged VLAN out the LAN (Ethernet) interface, those frames will be untagged. When WMS5316 receives untagged traffic from the LAN (Ethernet) interface, those frames are assigned to the untagged VLAN.


If unchecked, WMS5316 tag all outgoing LAN (Ethernet) frames, and only accept incoming frames that are tagged with known VLAN IDs.


Note: The Untagged VLAN checkbox should only be unchecked if the hubs or switches on your LAN support the VLAN (802.1Q) standard. Likewise, the Untagged VLAN value should only be changed if the hubs/switches on your LAN support the VLAN (802.1Q) standard. Changing either of these values will result in a loss of IP connectivity if the hubs/switches on your network have not yet been configured with the corresponding VLANs. Also, the untagged VLAN should be unchecked when either WN802Tv2 or WG602v4 are managed, as those Access Points do not support 802.1Q Vlan Tagging.


Management VLAN

Management VLANs are used for managing traffic (Telnet, SNMP, and HTTP) to and from WMS5316.


Frames belonging to the Management VLAN are not given any 802.1Q header when sent over the trunk. If a port is in a single VLAN, it can be untagged. But if the port needs to be a member of multiple VLANs, it must be tagged.
